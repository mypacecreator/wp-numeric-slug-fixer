# WP Numeric Slug Fixer

WordPress の `%postname%` パーマリンク構造で、数字のみのスラッグが日付アーカイブにリダイレクトされる問題を修正するプラグイン。

## 問題の背景

`/%postname%/` パーマリンク設定時、`2023` や `42` のような数字だけのスラッグをもつ投稿は直接表示されない。WordPress のリライトルールが数字のURLセグメントを年・月・日アーカイブとして優先マッチするため、`/2023/` へのリクエストは年別アーカイブにリダイレクトされてしまう。

## 解決アプローチ

`wp_insert_post_data` フィルターで DB 書き込み前にスラッグを検査し、数字のみであればプレフィックスを付与して書き直す。デフォルトは `post-`（例: `2023` → `post-2023`）。

「保存後に再更新」ではなく「保存前にフィルター」なので、`wp_update_post()` の再呼び出しによる無限ループのリスクがない。

## 動作条件

| 項目 | バージョン |
|---|---|
| WordPress | 6.0 以上（動作確認: 6.9.x） |
| PHP | 7.4 以上（8.x も対応） |

## フック仕様

### フィルター: `wpnsf_prefix`

数字スラッグに付与するプレフィックスを変更できる。

```php
// デフォルト: 'post-'
add_filter( 'wpnsf_prefix', function() {
    return 'article-';
} );
```

プレフィックス変更後は既存の数字スラッグを持つ投稿を再保存する必要がある（フィルターは保存時のみ実行されるため）。

## 開発メモ

### ローカル環境セットアップ

wp-env または Local などの WordPress 開発環境を用意し、このディレクトリを `/wp-content/plugins/wp-numeric-slug-fixer/` に配置してプラグインを有効化する。

### 手動テスト手順

1. 設定 > パーマリンク で「投稿名」を選択して保存
2. 新規投稿を作成し、スラッグを `2023` などの数字のみに設定
3. 保存後、スラッグが `post-2023` に変わっていることを確認
4. 実際の URL にアクセスして投稿が表示されることを確認（日付アーカイブにリダイレクトされないこと）

### 考慮済みのエッジケース

| ケース | 挙動 |
|---|---|
| スラッグが空 | WordPress が後でタイトルから生成するためスキップ |
| リビジョン投稿 | リビジョンは独自のスラッグ処理があるためスキップ |
| `%postname%` を含まないパーマリンク | 問題が発生しないため処理をスキップ |
| 数字を含むが数字のみでないスラッグ（例: `2023-recap`） | `ctype_digit()` が false を返すためスキップ |
